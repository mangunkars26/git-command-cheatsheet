Git Commands for Web Blog Development

1. Initialize a Git Repository

To start tracking your blog project with Git, initialize a new repository.

git init
Example: You have created a new project called "My Blog" and want to initialize Git. 
git init

2. Add Files to Staging Area

Add your HTML, CSS, and other files to the staging area before committing them.

git add 
            git add .
Example: You have just added an index.html and a style.css for your blog's homepage: 
git add index.html style.css
Or you can add all changes: 
git add .

3. Commit Your Changes

After staging your files, commit them with a meaningful message describing the changes.

git commit -m "Your message"
Example: You have finished the basic structure of your blog's homepage: 
git commit -m "Add homepage with basic HTML and CSS"

4. Check the Status of Your Repository

Use this command to check the current state of your project files (which are modified, staged, or untracked).

git status
Example: You've added a new file about.html for your blog's About page but haven't added it to Git yet. You can check the status: 
git status

5. View Commit History

To see a log of all your past commits.

git log
Example: You've been working on your blog for a while and want to see the history of your commits: 
git log --oneline

6. Create a New Branch

Use branches to add new features to your blog without affecting the main codebase.

git branch 
            git checkout
Example: You want to add a "Contact" page to your blog, so you create a new branch for it: 
git checkout -b feature/contact-page

7. Merge Branches

After completing work on a feature branch, you can merge it back into the main branch.

git checkout 
            git merge
Example: After finishing the "Contact" page on the feature/contact-page branch, merge it into the main branch: 
git checkout main
            git merge feature/contact-page

8. Discard Unstaged Changes

If you've made changes to a file but want to revert it to the last committed state.

git checkout --
Example: You accidentally made changes to style.css but want to discard those changes: 
git checkout -- style.css

9. Remove a Branch

After merging a branch, you can delete it if it's no longer needed.

git branch -d
Example: After merging the feature/contact-page branch, delete it: 
git branch -d feature/contact-page

10. Connect to a Remote Repository

To upload your blog project to GitHub or another remote repository, first connect your local repository to the remote.

git remote add origin
Example: You're ready to push your blog project to GitHub: 
git remote add origin https://github.com/username/my-blog.git

11. Push to a Remote Repository

Upload your local commits to a remote repository like GitHub.

git push origin
Example: Push the latest changes of your blog to the main branch on GitHub: 
git push origin main

12. Pull Changes from Remote Repository

Fetch and merge changes from a remote repository to your local repository.

git pull origin
Example: You want to pull the latest changes from the main branch on GitHub: 
git pull origin main

13. Clone a Remote Repository

To download a copy of a remote repository to your local machine.

git clone
Example: You want to clone your blog project from GitHub: 
git clone https://github.com/username/my-blog.git

14. View Differences Between Commits

See the differences between your local changes and the last commit.

git diff
Example: You want to see the differences in the about.html file before committing: 
git diff about.html

15. Undo the Last Commit

If you want to undo the last commit but keep your changes in the working directory.

git reset --soft HEAD~1
Example: You realize your last commit message was wrong, so you reset the commit: 
git reset --soft HEAD~1

16. Force Push

To overwrite the remote repository's history with your local changes.

git push origin  --force
Example: You've rewritten the commit history of your blog and need to force push to GitHub: 
git push origin main --force

17. Stash Changes

Temporarily save your uncommitted changes without committing them.

git stash
            git stash pop
Example: You're working on styling, but need to switch to another task without committing: 
git stash
Later, you can retrieve the changes with: 
git stash pop
